Climate-health effects in TRIGGER wearable data

Exploratory within-user physiological deviations during thermal exposure

Aim

This notebook investigates whether daily environmental heat exposure is associated with short-term physiological deviations measured by wearable devices. Because wearable measurements are highly subject-specific, the main analysis focuses on within-user z-scores, i.e. daily deviations from each participant’s own baseline.

The notebook deliberately avoids a strict complete-case filter at the beginning. Missing physiological values are retained in the master table and removed only within each outcome-specific analysis.

z_safe <- function(x) {
  ok <- !is.na(x)

  if (
    sum(ok) < 2 ||
    is.na(sd(x, na.rm = TRUE)) ||
    sd(x, na.rm = TRUE) == 0
  ) {
    return(rep(NA_real_, length(x)))
  }

  as.numeric((x - mean(x, na.rm = TRUE)) / sd(x, na.rm = TRUE))
}

safe_t_test_p <- function(x, g) {
  keep <- !is.na(x) & !is.na(g)
  x <- x[keep]
  g <- g[keep]

  if (length(unique(g)) < 2 || any(table(g) < 2)) {
    return(NA_real_)
  }

  tryCatch(stats::t.test(x ~ g)$p.value, error = function(e) NA_real_)
}

5. Within-user physiological deviations

Each physiological variable is converted to a within-user z-score. A value of 1 means that the subject was one standard deviation above their own average on that day. This controls for stable between-subject differences and focuses on daily deviations from the participant’s usual physiological baseline.

wearable_z <- wearable_daily |>
  group_by(userId) |>
  mutate(across(all_of(phys_vars), z_safe, .names = "{.col}_z")) |>
  ungroup() |>
  group_by(city) |>
  mutate(
    t2m_city_z = z_safe(t2m_daily_mean),
    utci_city_z = z_safe(utci_daily_mean)
  ) |>
  ungroup() |>
  mutate(month = floor_date(date, "month"))

6. Heatwave physiological fingerprint

This analysis compares heatwave and non-heatwave days for each physiological variable using all available observations for that variable. Therefore, the number of observations may differ across outcomes.

7. Outcome-specific mixed-effect models

After the fingerprint analysis, I used outcome-specific mixed models to better understand the observed differences. The fingerprint gives the overall heatwave vs non-heatwave contrast, but part of that contrast may actually be due to continuous thermal exposure, activity level or sleep duration.

For each physiological outcome, the model includes the heatwave indicator together with UTCI, steps and sleep duration. This allows the heatwave coefficient to be interpreted as the residual association with heatwave days after considering these covariates. The other coefficients are also useful, because they indicate whether the physiological outcome is more directly related to temperature, movement or sleep.

The random intercept for userId accounts for the repeated daily observations collected from the same participants.

fit_heatwave_lm <- function(data, outcome,
                            adjust_activity = TRUE,
                            adjust_sleep = TRUE) {
  outcome_z <- paste0(outcome, "_z")
  candidate_covars <- c("heatwave", "utci_city_z")

  if (adjust_activity && "steps_z" %in% names(data) && outcome != "steps") {
    candidate_covars <- c(candidate_covars, "steps_z")}

  if (adjust_sleep && "sleepduration_z" %in% names(data) && outcome != "sleepduration") {
    candidate_covars <- c(candidate_covars, "sleepduration_z")}

  model_vars <- c(outcome_z, candidate_covars, "userId")

  dat <- data |> select(all_of(model_vars)) |> drop_na()

  if (nrow(dat) < 50 || length(unique(dat$userId)) < 3 || length(unique(dat$heatwave)) < 2) {
    return(NULL)}

  form <- as.formula(paste0(outcome_z, " ~ ", paste(candidate_covars, collapse = " + ")))
  fit <- stats::lm(form, data = dat)
  attr(fit, "model_formula") <- deparse(form)
  attr(fit, "n_obs") <- nobs(fit)
  
  fit
}
